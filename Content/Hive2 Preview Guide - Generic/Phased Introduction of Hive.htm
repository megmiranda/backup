Phased Introduction of Hive 2 in your environment

The following outline summarizes a possible Hive 2 rollout plan. The goals in this are:

  • Minimizing disruption to workloads currently running successfully in Hive 0.13.
  • Maximizing the benefits from Hive 2.x in timeframes appropriate to the business.

 

 

Date

Major steps

Preview stage

TD enables  Hive 2 preview release for the customer’s account. Once enabled, both Hive 2 and Hive 0.13 will be available through TD Toolbelt CLI, and REST API, but not the console yet (we will update this doc when that becomes available).

 

Until GA, Hive 2 is not recommended for use in production workloads, but it should be generally representative of production features, performance, and quality.

 

The option provided to run queries with hive2 is most appropriate to use in testing, while modifying existing queries to remove hive2 incompatible syntax and make queries runnable in both Hive 0.13 and Hive 2.

 

Work with technical account management or support on testing new and existing workloads for use with Hive 2, based on this document.

 

A good goal is to rewrite many/most queries to use the stricter Hive 2-compatible syntax and run the modified queries in Hive 0.13 until GA, at which time you can fully switch your workload to Hive 2.

 

When creating new queries, avoid the incompatible syntax forms from Hive 0.13 described in “Known Backward Incompatibilities” section of this document.

 

At the time of writing this doc, an engineer can run a query with Hive 0.13 using normal methods such as  ‘td query -t hive’  and run a query with Hive 2 using  ‘td query -t hive --engine-version=experimental’  option.

 

During the beta period, we expect you to use Hive2 with staging databases and/or environments. While we have confidence in the current quality of the Hive2 release, and will work with you on beta activities and any experiments you wish to do in staging, we do not recommend or support the use of this experimental option for production workloads. 

 

If you finish your preview activities before Hive2 is GA (available for production workloads for all customers) then based on our mutual level of confidence in the software, we can discuss supporting production use before the official GA date.

 

 

 

2019-06

Hive 2 released to production.

  • Customers can use Hive 2 for production workloads.
  • By default, queries will still run in Hive 0.13 if no version is specified.
  • TD recommends the use of Hive 2 for new ad-hoc and batch workloads
  • Review and upgrade of existing workloads continues.

Switch Default Date

At a date of the customer’s choosing, Hive 2 becomes the default Hive release for the customer’s account.

 

Users will still have the option of running queries with  Hive 0.13  from the TD Console, API, and CLI, but the default will change to hive2.

 

The TD Console will show Hive 2 as the default Hive version. In CLI and API, if no version is specified, then Hive 2 will be used.

 

After this date, Hive queries that show unexpected compatibility issues can still run in Hive 0.13.
