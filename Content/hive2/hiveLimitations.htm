Limitations on Custom Hive and MapReduce Code Support

There are limitations on DDL in TD Hive, as there are in TD Presto because of our flexible schema.

 

Treasure Data limits the use of custom code in our Hadoop and Hive environment because we do not provide dedicated Hive clusters for individual customers. The following limitations are a result of that:

  • Custom Hive UDFs cannot be added, because these UDFs would be exposed for use by all customers.
  • Custom MapReduce scripts cannot be added due to security and system stability considerations.

Alternatives for running custom code within Arm TD include:

  • running custom code in Docker containers, see Custom Scripts.
  • running Spark workloads using our TD-Spark capability to access data in the data lake, see Launcing TD-Spark.
