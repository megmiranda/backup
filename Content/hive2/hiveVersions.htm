About TD Hive Supported Versions

The following versions are supported:

  • TD Hive 2019.1 and later
  • TD Hive 0.13

TD Hive 2019.1 and later

  • Is based on the Hive releases in active development by the open source community. ANSI compatible, as well as mostly forward compatible with Hive 0.13. We recommend that all new data pipeline and ad hoc Hive workloads run in these releases. These newer releases include many enhancements that significantly enhance your ability to work with your data. For example:
    • stability enhancements
    • improved analysis of query issues
    • query optimizations
    • support for Treasure Data’s user-defined partitioning
    • Tez runtime
    • enterprise data security model
    • bug fixes from the open source community

     

    See upgrading to Hive 2 for details on how to clean up your syntax for a smooth TD Hive 2019 migration.

    TD Hive 0.13

     

    A Treasure Data-built release of Hive that backported many features and bug fixes from more recent Hive releases to the 2014 Apache Hive 0.13 release. Hive 0.13 uses HiveQL, a SQL-like query language that is mostly a subset of ANSI SQL, with a few non-standard extensions. TD Hive 0.13 also added some Treasure Data-specific features such as the now-deprecated v[] column syntax.

     

    Continues to be a fully supported Hive release in our environment for the foreseeable future. However, the open source community no longer provides fixes and enhancements to Hive 0.13, and backporting updates is impractical because the Hive code base is too different. Therefore we will limit further development and backporting investment in our Hive 0.13-based offering.
